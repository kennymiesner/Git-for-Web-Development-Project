    1. What is Semantic HTML? 
        HTML that gives meaning to our markup.
    2. What is HTML used for? 
        HTML is used to build the structure of the page.
    3. What is an attribute and where do we put it? 
        Extension of an element to change behavior.
    4. What is the h1 tag used for? How many times should I use it on a page?
        H1 is used for the page title and is the most important heading on the page. This should only be used once.
    5. Name two tags that have required attributes.
        <img> and <a>
    6. What do we put in the head of our HTML document? 
        Metadata such as document title, styles, scripts, etc.
    7. What is an id? 
        Specifies a unique id for an element. It can only be used once on a page.
    8. What elements can I add an id to? 
        All HTML elements
    9. How many times can I use the same id on a page? 
        Only once per page.
    10. What is a class? 
        Used to point to a class in a style sheet. 
    11. What elements can I add a class to? 
        All HTML elements
    12. How many times can I use the same class on a page? 
        As many as you want.
    13. How do I get my link to open in a new tab?
        target="_blank"
    14. What is the alt attribute in the image tag used for? 
        Descriptive text typically used for accessibility purposes and screen readers.
    15. How do I reference an id?
        #id-name
    16. What is the difference between a section and a div?
        Section is semantic, while a div is not.
    17. What is CSS used for? 
        CSS is used to style the HTML or structure of the page. If HTML is the homes foundation and framework, CSS is the paint, flooring, etc.
    18. How do we select an element? Example - every h2 on the page
        In this case, we would want to use h2 { }
    19. What is the difference between a class and an id? - Give me an example of when I might use each one?
        Main difference is class can be used multiple times on a page, while id should only be used once. While both can be used to target styles, id is also used for anchoring purposes. Example: .blue { } (Apply to any HTML element) #section-01 (Should only be used once to anchor to the section.)
    20. How do we select classes in CSS?
        .class { }
    21. How do we select a p element with a single class of “human””?
        p.human { }
    22. What is a parent child selector? When would this be useful? 
        A parent child selector is a way to target an element within a parent element. You could use this to target the p elements within a specific section with, .main p { }
    23. How do you select all links within a div with the class of sidebar?
        .sidebar a { }
    24. What is a pseudo selector?
        Used to define a specific state of an element, such as :hover or :active.
    25. What do we use to change the spacing between lines?
        line-height: ;
    26. What do we use to change the spacing between letters?
        letter-spacing: ;
    27. What do we use to to change everything to CAPITALS? lowercase? Capitalize?
        text-transform: uppercase; text-transform: lowercase; text-transform: capitalize;
    28. How do I add a 1px border around my div that is dotted and black?
        div {border: 1px dotted black;}
    29. How do I select everything on the page? 
        *
    30. How do I write a comment in CSS?
        /* This is a comment */
    31. How do I find out what file I am in, when I am using the command line? 
        pwd
    32. Using the command line - how do I see a list of files/folders in my current folder?
        ls
    33. How do I remove a file via the command line? Why do I have to be careful with this? 
        rm, it will delete the file without prompts.
    34. Why should I use version control? 
        Version control should be used to keep previous versions of a file in the chance that you may have to revert back to a previous version, i.e. a bug is introduced somewhere along the way.
    35. How often should I commit to github?
        Often. Every ~20 minutes or if a new addition is completed.
    36. What is the command we would use to push our repo up to github? 
        $ git push -u origin branch-name
    37. Walk me through Lambda's git flow. 
        a. Fork repo.
        b. Double-check that your username is in the URL and not Lambda's.
        c. Clone the repo by first clicking the "Clone or download" button and then the clipboard icon next to the HTTPS URL.
        d. In Terminal, cd into the directory that you want to clone to. In our case, we've been using the Desktop.
        e. Enter git clone <link-to-repo>
        f. Then, cd into the newly cloned repo with cd <repo-name>
        g. Use git checkout -b 'firstName-lastName' to branch the repo
        h. Make edits to any files needed for the project.
        i. Once changes are complete, go back to Terminal and use git add . to add edited files to the staging area
        j. Use git commit -m "Details of updates made" 
        k. Push changes to GitHub repo with git push -u origin branch-name
        l. Go back to the forked project in GitHub and submit a pull request

Stretch Questions

    1. What is the difference between an inline element and a block element?
        An inline element only takes as much space as needed and does not start on a new line. Block elements take up the full-width available and start on a new line.
    2. What happens when an element is positioned absolutely? 
        The element is removed from the normal flow fo the document and does not take up space in the page layout. Positioned relative to its closest ancestor and positioned using top, right, bottom, left values. 
    3. How do I make an element take up only the amount of space it needs but also have the ability to give it a width?
        inline-block
    4. Name 3 elements that are display block by default, 2 elements that are display inline by default and 1 element that is display inline-block by default
        Display block: div, p, section
        Display inline: span, a
        Display inline-block: img
    5. In your own words, explain the box model. What is the "fix" for the box model, in other words, how do we make all elements respect the width we've given them? 
        Model used to explain the basis of every HTML element on the page that consists of the margin, border, padding, and content, each affecting the final size of the element. To fix, you could use box-sizing: border-box;
